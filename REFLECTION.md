Fluento - Build Reflections
What went well
Getting the core AI loop working was smoother than I expected. The Groq API was super fast, and the structure of its responses (JSON) made it easy to get scores and corrections. Setting up Supabase auth was also easy — the RLS policies and onAuthStateChange listener covered most of the logic for handling authentication. Once I got these pieces to work, everything else fell into place easily.
What didn't go well
Voice input was where I hit the most friction. The browser MediaRecorder API posed several challenges, such as making sure that the microphone stops once you finish recording. Another challenge was the async hand-off of audio data from the media recorder -> Whisper -> getting a response from the AI. Lastly, implementing the streak system wasn't as straightforward as I anticipated. Supabase's upsert with onConflict doesn't always behave as expected, so I opted for a more manual solution.
The app UI went through several complete overhauls. The early iterations had too much of the desktop feel, as well as feeling too generic. Getting the interface to look like an actual mobile application (with bottom nav, accessible microphone button, and visualized AI response) took considerably longer than developing the app’s core features.
Changes that would have been made given more time
The Whisper API request would have been moved to serverless function, similar to the one for the tutor API, so as to hide neither the Whisper nor the tutor API key in the client-side code. Moreover, the pronunciation assessment would be calculated using an API specifically designed for that purpose (such as Azure Speech), not derived from the grammar test results as was done in the current version. The word list for the vocab quiz is static, and ideally, it should change depending on what mistakes were made in practice sessions to create a closed training circuit involving speaking and vocab learning.
